## 🌱 Qu'est-ce que BiotechProject ?

BiotechProject est un **laboratoire numérique ouvert** qui combine **biotechnologie, santé et développement web** dans un système intégré conçu pour être :

- 🔍 **Scientifiquement fiable**
- 💻 **Techniquement robuste**
- 🌐 **Accessible à tous**, y compris les utilisateurs handicapés
- 🤝 **Ouvert à la collaboration mondiale**

Il a été créé pour démontrer comment la technologie peut servir la science et la santé, tout en garantissant un accès équitable pour tous – indépendamment des capacités sensorielles, cognitives ou motrices.

C'est un espace collaboratif pour développeurs, chercheurs et passionnés qui veulent explorer **comment le web peut devenir un outil d'inclusion et d'innovation scientifique**.

## 🎯 Public Cible et Vision Stratégique

BiotechProject est conçu pour les parties prenantes qui considèrent la résilience comme une exigence clinique :

* **Systèmes de Santé et Initiatives de Santé** : Conçu pour mettre à l'échelle des jumeaux numériques métaboliques avec des coûts de calcul zéro côté serveur, en tirant parti de Vanilla JS haute performance pour une logique complexe côté client.

* **Patients Neurodivergents** : Conformité native au **WCAG 2.2 AAA Gold Standard** et 'Mode Confort' offrent un environnement cognitivement sûr pour l'interaction santé, réduisant la surcharge sensorielle.
* **SRE et Ingénieurs Systèmes** : Un blueprint pour "SRE-for-Humans", où les métriques de performance (score agrégé 95%) sont traitées comme un engagement pour l'inclusion et la fiabilité des utilisateurs.

### [ADR-001] Mandat Zero-Framework
* **Décision** : Éviter les frameworks modernes (React/Angular/Vue) en faveur de pure **Vanilla JS (ES6+)**.
* **Justification** : Élimine le "taxe framework". Dans les contextes critiques pour la santé, l'interactivité sub-seconde est une exigence clinique.
* **Résultat** : Taille initiale du bundle **< 20KB** et TTI de **0.3s**.
* **Validation** : [View Engineering Log (#12)](https://github.com/Gitechnolo/biotechproject/issues/12)

### [ADR-002] Stateless Edge & Intelligence Côté Client
* **Décision** : Migrer toute la logique complexe de synchronisation métabolique et biologique côté client.
* **Justification** : Garantit **100% disponibilité de service** pendant les pics réseau ou les pannes backend.
* **Résultat** : 85% réduction des coûts d'infrastructure.
* **Validation** : [View Resilience Report (#13)](https://github.com/Gitechnolo/biotechproject/issues/13)

### [ADR-003] Audit SRE Assisté par IA
* **Décision** : Orchestration multi-modèle (Gemini + Copilot) pour la révision de code.
* **Justification** : Surmonte les hallucinations de LLM individuels via "consensus" sur la sécurité des types.
* **Résultat** : 87% score de maturité technique.
* **Validation** : [View Audit Protocol (#14)](https://github.com/Gitechnolo/biotechproject/issues/14)

### [ADR-004] Machine d'État Circadienne
* **Décision** : Implémentation du pattern efficace en ressources **Early Return**.
* **Justification** : Réduit la charge cognitive sans déclencher des re-renders coûteux ou un polling lourd.
* **Résultat** : 98% réduction des cycles CPU en arrière-plan.
* **Validation** : [View Performance Pattern (#15)](https://github.com/Gitechnolo/biotechproject/issues/15)

### [ADR-005] Portabilité de Données Orientée SRE
* **Décision** : Déploiement d'une couche d'exportation zero-latence utilisant les APIs natives Blob.
* **Justification** : Assure la souveraineté des données pour les dossiers de santé avec "Lean Logic" (pas de bibliothèques).
* **Résultat** : Génération instantanée CSV/JSON.
* **Validation** : [View Export Strategy (#17)](https://github.com/Gitechnolo/biotechproject/issues/17)

### [ADR-006] AAA Accessibility Baseline
* **Décision** : Implémentation de conformité **WCAG 2.2 Level AAA Gold Standard**, incluant Enhanced Focus Appearance et des standards de support cognitif.
* **Justification** : Éléver l'inclusion digitale du logiciel clinique au niveau le plus élevé, en assurant sécurité cognitive et optimisation sensorielle.
* **Résultat** : Conformité totale avec WCAG 2.2.
* **Validation** : [View Accessibility Statement (#19)](https://github.com/Gitechnolo/biotechproject/issues/19)

### [ADR-007] Audit WCAG 2.2 AAA Resilient (Stateless Edge)
* **écision**: Audit WCAG 2.2 AAA Stateless Edge
* **Justification**: L'approche "Zero-Framework" élimine les bibliothèques externes lourdes, tandis que le "Semantic Snapshotter" capture efficacement l'état de l'interface, réduisant la dépendance à l'infrastructure et améliorant l'équité sanitaire (Health Equity) dans les régions à faible connectivité.
* **Résultat**: Audit d'accessibilité ultra-léger (~2KB) via Copilot, garantissant le fonctionnement dans les environnements cliniques à faible bande passante.
* **Validation** : [View Issue #20](https://github.com/Gitechnolo/biotechproject/issues/20)

### [ADR-008] Infrastructure: Nightly SRE Reliability Gate & Immutable Integrity
* **Décision**: Mise en œuvre d'un "Nightly SRE Reliability Gate" automatisé.
* **Justification**: Garantit que les scores Lighthouse à 100% et la conformité WCAG 2.2 AAA ne subissent aucune régression. Dans les environnements cliniques, l'intégrité logicielle (Immutable Integrity) est une exigence de sécurité critique pour la résilience du système.
* **Résultat**: Cycle de validation automatisé de 24h avec "Zero-Tolerance" pour la dégradation des performances ou de l'accessibilité.
* **Validation** : [View Issue #21](https://github.com/Gitechnolo/biotechproject/issues/21)

### [ADR-009] Resilient Edge : Asset Pruning & GPU Stabilization
* **Décision** : Implémentation d'un "Clinical Accessibility Mode" automatisé, déclenché lorsque le RTT > 1000ms.
* **Justification** : Les arrière-plans haute résolution et les animations 3D (hélice d'ADN) consomment une bande passante critique et des cycles CPU lors de pics de latence réseau.
* **Résultat** : Récupération instantanée de 70KB de bande passante et 0% de surcharge GPU via des instantanés CSS statiques.
* **Validation** : [View Issue #23](https://github.com/Gitechnolo/biotechproject/issues/23)

### [ADR-010] Convergence Neurale : Modélisation Prédictive Multisectorielle
* **Décision** : Implémentation du **Neural Inference Engine (v6.1)** pour l'analyse intersectorielle.
* **Justification** : Évoluer au-delà de la résilience statique vers la prévision proactive, en fusionnant les jeux de données de l'OMS (Bio) et de Gartner (Cyber).
* **Résultat** : Synchronisation en temps réel entre le canvas accéléré par GPU et le DOM sémantique, garantissant l'accessibilité de tous les points de données prédictifs.
* **Validation** : [Voir la Logique Neurale (#26)](https://github.com/Gitechnolo/biotechproject/issues/26)

### [ADR-011] Neural Core v6.3.2 : Gardien SRE Prédictif et Système Immunitaire Bio-Numérique
* **Décision** : Implémentation de `BiotechCoreWorker.js` (Moteur d'Inférence Neuronale) et `BiotechGuardian.js` (Capteur Ambiant).
* **Rationale** : Évoluer d'une remédiation réactive vers une **Résilience Prédictive**. En déléguant l'inférence de charge neuronale à un Web Worker dédié, le système prévoit les dégradations de performance (TBT/Long Tasks) avant qu'elles n'affectent l'interface.
* **Résultat** : Une réponse immunitaire bio-numérique à l'échelle autonome, vérifiant une **réduction du TBT à 112ms** et appliquant la politique d'**Oubli Éthique des Données** (purge tous les 7 jours).
* **Validation** : [Voir l'Évolution Neurale et le Rapport de Benchmark (#27)](https://github.com/Gitechnolo/biotechproject/issues/27#issuecomment-4288960068)

## 🌐 Accessibilité

Le site est **conforme au WCAG 2.2 Level AAA Gold Standard** pour toutes les pages principales.  
La conformité a été vérifiée par :

- Audits automatisés (Lighthouse, axe, WAVE)
- Tests manuels avec lecteurs d'écran (NVDA, VoiceOver)
- Navigation clavier complète (tab, shift+tab, enter, space, arrows)
- Validation du code W3C
- Inspection directe du code pour structure sémantique et utilisation correcte d'ARIA

Le projet est **partiellement conforme à Level AAA**, notamment dans :
- Contraste des couleurs (la plupart des textes dépassent 7:1)
- Structure hiérarchique des titres
- Utilisation de texte alternatif descriptif

Cependant, certains critères AAA ne sont pas applicables ou requis dans le contexte actuel (par ex. sous-titres vidéo, langage simple étendu).

## 🌍 Gestion Multilingue (i18n)

BiotechProject prend en charge **plusieurs langues** grâce à un système de traduction **modulaire, léger et accessible**, conçu pour fonctionner de manière transparente sur des pages statiques hébergées sur GitHub Pages.
"Alors que le contenu textuel est entièrement localisé, les diagrammes scientifiques privilégient actuellement les versions italien/anglais pour maintenir la précision technique pendant l'évolution rapide du projet."

Le système permet :
- ✅ Traduction de contenu en temps réel  
- ✅ Persistance de la sélection de langue entre les pages (comme Wikipedia ou Google)  
- ✅ Support pour des versions simplifiées pour les utilisateurs dyslexiques  
- ✅ Extension facile par les contributeurs  

### 🧩 Architecture du Système

- **Fichiers JSON modulaires** : chaque page a son propre fichier de traduction dans le répertoire `lang/`  
- **Common.json** : contient des textes partagés (menu, footer, bouton langue)  
- **Aucun backend requis** : tout fonctionne en JavaScript pur  
- **LocalStorage** : se souvient de la langue sélectionnée par l'utilisateur  
- **`data-lang-key`** : attribut HTML pour identifier les éléments traduisibles  

### 📁 Structure du dossier `lang/`

`common.json` contient des chaînes partagées (menu, footer, etc.), les fichiers de page contiennent du texte par page.

```text
lang/
├── common.json                  # Shared strings (menu, footer, UI)
│
├── home.json                    # index.html
├── progetti.json                # Progetti.html
├── staff.json                   # Staff.html
├── marketing.json               # Marketing.html
├── tech_maturity.json           # Tech_Maturity.html
│
├── dermatologia.json            # Dermatologia.html + Dermatologia-semplice.html
├── cuore.json                   # Cuore.html + Cuore-semplice.html
├── cellula.json                 # Cellula.html + Cellula-semplice.html
├── apparato_digerente.json      # Apparato_digerente.html + -semplice.html
├── apparato_respiratorio.json   # Apparato_respiratorio.html + -semplice.html
├── apparato_tegumentario.json   # Apparato_tegumentario.html + -semplice.html
├── sistema_linfatico.json       # Sistema_linfatico.html + -semplice.html
└── capelli.json                 # Capelli.html + Capelli-semplice.html
```
