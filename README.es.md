## 🌱 ¿Qué es BiotechProject?

BiotechProject es un **laboratorio digital abierto** que combina **biotecnología, salud y desarrollo web** en un sistema integrado diseñado para ser:

- 🔍 **Científicamente confiable**
- 💻 **Técnicamente robusto**
- 🌐 **Accesible para todos**, incluyendo usuarios con discapacidades
- 🤝 **Abierto a la colaboración global**

Fue creado para demostrar cómo la tecnología puede servir a la ciencia y la salud, garantizando acceso equitativo para todos – independientemente de habilidades sensoriales, cognitivas o motoras.

Es un espacio colaborativo para desarrolladores, investigadores y entusiastas que quieren explorar **cómo la web puede convertirse en una herramienta de inclusión e innovación científica**.

## 🎯 Público Objetivo y Visión Estratégica

BiotechProject está diseñado para stakeholders que consideran la resiliencia como un requisito clínico:

* **Sistemas de Salud e Iniciativas de Salud**: Diseñado para escalar gemelos digitales metabólicos con costos cero de computación del lado del servidor, aprovechando Vanilla JS de alto rendimiento para lógica compleja del lado del cliente.
* **Equidad en Salud Global**: Diseñado específicamente para usuarios en regiones de baja conectividad, garantizando tiempos de carga sub-segundo (0,3s) incluso en dispositivos legacy gracias a optimización SRE-grade.
* **Pacientes Neurodivergentes**: Cumplimiento nativo WCAG AAA y 'Modo Confort' ofrecen un entorno cognitivamente seguro para interacción en salud, reduciendo sobrecarga sensorial.
* **SRE e Ingenieros de Sistemas**: Un blueprint para "SRE-for-Humans", donde las métricas de rendimiento (puntuación agregada 95%) se tratan como un compromiso para inclusión y fiabilidad del usuario.

### [ADR-001] Mandato Zero-Framework
* **Decisión**: Evitar frameworks modernos (React/Angular/Vue) en favor de puro **Vanilla JS (ES6+)**.
* **Justificación**: Elimina el "impuesto de framework". En contextos críticos para la salud, la interactividad sub-segundo es un requisito clínico.
* **Resultado**: Tamaño inicial del bundle **< 20KB** y TTI de **0.3s**.
* **Validación**: [View Engineering Log (#12)](https://github.com/Gitechnolo/biotechproject/issues/12)

### [ADR-002] Stateless Edge & Inteligencia del Lado del Cliente
* **Decisión**: Migrar toda la lógica compleja de sincronización metabólica y biológica al lado del cliente.
* **Justificación**: Garantiza **100% disponibilidad de servicio** durante picos de red o fallos de backend.
* **Resultado**: 85% reducción en costos de infraestructura.
* **Validación**: [View Resilience Report (#13)](https://github.com/Gitechnolo/biotechproject/issues/13)

### [ADR-003] Auditoría SRE Asistida por IA
* **Decisión**: Orquestación multi-modelo (Gemini + Copilot) para revisión de código.
* **Justificación**: Supera alucinaciones de LLMs individuales vía "consenso" en seguridad de tipos.
* **Resultado**: 87% puntuación de madurez técnica.
* **Validación**: [View Audit Protocol (#14)](https://github.com/Gitechnolo/biotechproject/issues/14)

### [ADR-004] Máquina de Estado Circadiana
* **Decisión**: Implementación del patrón eficiente en recursos **Early Return**.
* **Justificación**: Reduce carga cognitiva sin disparar re-renders costosos o polling pesado.
* **Resultado**: 98% reducción de ciclos CPU en background.
* **Validación**: [View Performance Pattern (#15)](https://github.com/Gitechnolo/biotechproject/issues/15)

### [ADR-005] Portabilidad de Datos Orientada a SRE
* **Decisión**: Despliegue de una capa de exportación zero-latencia usando APIs nativas Blob.
* **Justificación**: Asegura soberanía de datos para registros de salud con "Lean Logic" (sin bibliotecas).
* **Resultado**: Generación instantánea CSV/JSON.
* **Validación**: [View Export Strategy (#17)](https://github.com/Gitechnolo/biotechproject/issues/17)

## 🌐 Accesibilidad

El sitio es **conforme con WCAG 2.1 Level AA** para todas las páginas principales.  
La conformidad ha sido verificada mediante:

- Auditorías automatizadas (Lighthouse, axe, WAVE)
- Pruebas manuales con lectores de pantalla (NVDA, VoiceOver)
- Navegación completa por teclado (tab, shift+tab, enter, space, arrows)
- Validación de código W3C
- Inspección directa del código para estructura semántica y uso correcto de ARIA

El proyecto es **parcialmente conforme con Level AAA**, particularmente en:
- Contraste de color (la mayoría de los textos supera 7:1)
- Estructura jerárquica de títulos
- Uso de texto alternativo descriptivo

Sin embargo, algunos criterios AAA no son aplicables o requeridos en el contexto actual (ej. subtítulos de video, lenguaje simple extendido).   

📄 **Declaración completa de accesibilidad:**

## 🌍 Gestión Multilingüe (i18n)

BiotechProject soporta **múltiples idiomas** a través de un sistema de traducción **modular, ligero y accesible**, diseñado para funcionar de manera transparente en páginas estáticas alojadas en GitHub Pages.
"Mientras el contenido textual está completamente localizado, los diagramas científicos priorizan actualmente versiones italiano/inglés para mantener precisión técnica durante la evolución rápida del proyecto."

El sistema permite:
- ✅ Traducción de contenido en tiempo real  
- ✅ Persistencia de la selección de idioma entre páginas (como Wikipedia o Google)  
- ✅ Soporte para versiones simplificadas para usuarios disléxicos  
- ✅ Extensión fácil por contribuidores  

### 🧩 Arquitectura del Sistema

- **Archivos JSON modulares**: cada página tiene su propio archivo de traducción en el directorio `lang/`  
- **Common.json**: contiene textos compartidos (menú, footer, botón de idioma)  
- **Ningún backend requerido**: todo funciona en JavaScript puro  
- **LocalStorage**: recuerda el idioma seleccionado por el usuario  
- **`data-lang-key`**: atributo HTML para identificar elementos traductibles  

### 📁 Estructura de la carpeta `lang/`

`common.json` contiene cadenas compartidas (menú, footer, etc.), los archivos de página contienen texto por página.

```text
lang/
├── common.json                  # Shared strings (menu, footer, UI)
│
├── home.json                    # index.html
├── progetti.json                # Progetti.html
├── staff.json                   # Staff.html
├── marketing.json               # Marketing.html

├── dermatologia.json            # Dermatologia.html + Dermatologia-semplice.html

├── cellula.json                 # Cellula.html + Cellula-semplice.html
```
